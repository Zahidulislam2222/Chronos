# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| Latest (main branch) | Yes |

## Reporting a Vulnerability

If you discover a security vulnerability in Chronos, please report it responsibly.

**Email:** muhammadzahidulislam2222@gmail.com
**Subject line:** `SECURITY: Chronos — [brief description]`

### What to include

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

### Response timeline

- **72 hours** — acknowledgment of your report
- **30 days** — target for fix or mitigation

### What qualifies

- Authentication/authorization bypasses
- SQL injection, XSS, CSRF
- Sensitive data exposure
- GraphQL API security issues
- JWT token vulnerabilities

### Out of scope

- WordPress core vulnerabilities (report to [WordPress Security](https://hackerone.com/wordpress))
- Third-party plugin bugs (WooCommerce, ACF, WPGraphQL)
- Issues requiring physical access to the server
- Social engineering attacks

## Recognition

Valid reports will be credited in the fix commit and release notes (unless you prefer anonymity).
